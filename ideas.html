My Principles on UI Design

I've seen enough counterexamples of how software can be designed poorly. Here are some principles that I strive to follow when designing a UI:

  • Clear directions. A first-time user must be able to understand the next steps without time-consuming searching or random button-clicking.
  • Predictable behavior. The application should do exactly what the user instructed it to do, no more no less. Absolutely no auto-completing without their knowledge or approval.
  • Reversible actions. Users should be able to undo their actions. No more next button without a back button. No more accidental deletion without a way to restore the data.
  • Transparent Errors. Users should be able to understand why an error occurred, and how to fix it. No more "Something went wrong" or "Please try again later".
  • Lightweight. Things should not feel heavy. The UI should be as light as possible, and the user should be able to navigate through the UI with ease.
